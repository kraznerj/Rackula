Create `api/src/auth/config.ts` with Better Auth configuration using stateless (cookie-only) sessions:

- Import `betterAuth` from "better-auth"
- Configure session settings:
  - `expiresIn: 60 * 60 * 12` (12 hours, per research recommendation)
  - `updateAge: 60 * 60 * 6` (refresh when 6 hours remain)
  - `cookieCache.enabled: true` with `maxAge: 300` (5-minute cache)
- Configure cookie attributes:
  - `httpOnly: true` (XSS protection)
  - `sameSite: 'lax'` (CSRF protection)
  - `secure: true` (HTTPS only, auto-disabled in dev)
- Configure generic OIDC provider using environment variables:
  - `RACKULA_OIDC_ISSUER` (e.g., https://authentik.example.com/application/o/rackula/)
  - `RACKULA_OIDC_CLIENT_ID`
  - `RACKULA_OIDC_CLIENT_SECRET`
  - `RACKULA_OIDC_REDIRECT_URI` (defaults to /auth/callback)
- Export configured `auth` instance

Create `api/src/auth/client.ts` that exports the auth client for use in route handlers.

DO NOT configure database settings — omitting database config enables stateless mode per research. This eliminates the need for session store migration while providing sessions that survive restarts (stored in signed cookies, not server memory).

Create `api/src/middleware/auth.ts` with Better Auth integration for Hono:

1. Create `createAuthHandler()` function that wraps Better Auth's handler for Hono:
   - Accepts Hono Context
   - Returns `auth.handler(c.req.raw)` to pass Web Request to Better Auth
   - Handles both GET and POST methods for auth endpoints

2. Create `createAuthMiddleware()` function for protected routes:
   - Calls `auth.api.getSession({ headers: c.req.raw.headers })`
   - If no session and route requires auth, return 401
   - If session exists, attach to context: `c.set('authSession', session)`
   - For unauthenticated users accessing read-only routes (GET /api/layouts), allow through with no session
   - Return next() to continue

3. Create `createOptionalAuthMiddleware()` for routes that allow both authenticated and unauthenticated access:
   - Same as above but never returns 401
   - Attaches session if present, continues without session if not

Update `api/src/app.ts`:

1. Remove custom auth middleware imports from security.ts:
   - Remove `createAuthGateMiddleware`
   - Remove `resolveAuthenticatedSessionClaims`
   - Remove `createRefreshedAuthSessionCookieHeader`
   - Remove `invalidateAuthSession`
   - Keep CSRF and CORS middleware (independent concerns)

2. Import Better Auth handlers:

   ```typescript
   import {
     createAuthHandler,
     createOptionalAuthMiddleware,
   } from "./middleware/auth";
   import { auth } from "./auth/config";
   ```

3. Mount Better Auth routes:

   ```typescript
   app.on(["POST", "GET"], "/auth/*", createAuthHandler());
   app.on(["POST", "GET"], "/api/auth/*", createAuthHandler());
   ```

4. Replace custom auth gate middleware with optional auth middleware:
   - Apply `createOptionalAuthMiddleware()` globally
   - This allows unauthenticated access to read-only routes (preserving core value: "design with zero friction")

5. Keep existing write-auth token middleware for PUT/DELETE operations (separate from session auth)

6. Remove placeholder auth handlers:
   - Remove `authNotConfiguredHandler`
   - Remove manual `/auth/login`, `/auth/callback`, `/auth/check`, `/auth/logout` routes
   - Better Auth handles these automatically via mounted `/auth/*` routes

DO NOT modify routes/layouts.ts or routes/assets.ts yet — session access will be updated in a follow-up plan. Focus on core Better Auth integration in this task.

Reference patterns from RESEARCH.md "Hono Integration" section.
