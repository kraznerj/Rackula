  <action>
Update `api/src/auth/config.ts` to use the Better Auth genericOAuth plugin:

1. Add import: `import { genericOAuth } from "better-auth/plugins"`

2. Read environment variables:
   - `RACKULA_OIDC_ISSUER` — the OIDC provider's issuer URL
   - `RACKULA_OIDC_CLIENT_ID` — OAuth client ID
   - `RACKULA_OIDC_CLIENT_SECRET` — OAuth client secret
   - `RACKULA_OIDC_REDIRECT_URI` — callback URL (optional, Better Auth defaults to `{baseURL}/api/auth/oauth2/callback/{providerId}`)

3. Add `plugins` array to the betterAuth() config with genericOAuth:

```typescript
plugins: [
  genericOAuth({
    config: [
      {
        providerId: "oidc",
        clientId: process.env.RACKULA_OIDC_CLIENT_ID || "",
        clientSecret: process.env.RACKULA_OIDC_CLIENT_SECRET || "",
        discoveryUrl: process.env.RACKULA_OIDC_ISSUER
          ? `${process.env.RACKULA_OIDC_ISSUER.replace(/\/$/, '')}/.well-known/openid-configuration`
          : undefined,
        scopes: ["openid", "profile", "email"],
        pkce: true,  // Enhanced security for public clients
        redirectURI: process.env.RACKULA_OIDC_REDIRECT_URI || undefined,
      },
    ],
  }),
],
```

4. Remove the TODO comment block (lines 49-54) and replace with the actual plugin configuration.

5. Keep all existing session and cookie configuration unchanged.

6. Add `RACKULA_BASE_URL` env var reading for `baseURL` config (Better Auth needs this to construct callback URLs):

```typescript
baseURL: process.env.RACKULA_BASE_URL || "http://localhost:3000",
```

Update `api/src/auth/client.ts`:

1. Add import: `import { genericOAuthClient } from "better-auth/client/plugins"`
2. Add the plugin to the client: `plugins: [genericOAuthClient()]`
3. This enables `authClient.signIn.oauth2({ providerId: "oidc" })` on the client side.

DO NOT add a database configuration — stateless sessions remain the mode per user decision.
DO NOT modify middleware/auth.ts — it already handles sessions correctly.
DO NOT hard-code any IdP URLs — all configuration via environment variables.

The genericOAuth plugin uses `discoveryUrl` to auto-discover authorization, token, and userInfo endpoints from the OIDC provider's `.well-known/openid-configuration`. This works with Authentik, Authelia, Keycloak, and any OIDC-compliant provider.
